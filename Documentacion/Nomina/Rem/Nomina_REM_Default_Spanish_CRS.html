3 Contexto

3.1 Enunciado

Contexto de Operacion: El usuario manifiesta la necesidad de implementar una aplicacion que solucione de manera facil, agiìl, confiable y eficaz lo complejo que aveces parace ser realizar una nomina o los cobros de las actividades desarrolladas por el mismo

se pretende crear una aplicacion que permita la configuracion de un perfil con el fin de determinar si es un empleado o un independiente y apartir de esto establecer un valor de hora, valor dia, dias laborados y por ende un salario.

con esta aplicacion se puede determinar con antelacion un valor posible de nomina y asi el usuario podra organizar sus gastos y establecer su plan de ahorro.

Esta aplicacion permitira tener un alcance no solo personal sino tambien empresarial, pues permitira generar un informe completo de todos los datos de el usuario y posteriormente podra ser impresa al ser llevada a la computadora.

3.3 Caracteristicas


EL SISTEMA
-Administrar perfiles: El sistema permitira Crear,Actualizar,Borrar, Consultar.

CONSULTAS
-Realizar Consultas: (independientes) El usuario podrá realizar consultas desde su dispositivo movil de los pagos recibidos y horas laboradas

-(Empleado)- El usuario podra consultar los detalles de los pagos realizados en quincenas pasadas

AHORRO
-Generar valor ahorro : El usuario podra visualizar lo que corresponde al ahorro de acuerdo al valor devengado

INFORME
-Generar un informe de manera que en un solo pantallazo se puedan obtener lo mas relevante de la nomina.

Precondición Inicio del Sistema
Secuencia normal Paso Acción
1 El sistema Consulta si hay algun perfil creado
2 Si el sistema encuentra que no hay perfil creado, el sistema presenta la forma de creación del perfil
3 El actor Usuario (ACT-0001) ingresa sus datos personales.
4 El actor Usuario (ACT-0001) solicita al sistema guardar la información.
5 El sistema valida que toda la información esté completa
6 El sistema guarda la información y muestra un mensaje al usuario que la operación ha sido exitosa.
7 El sistema direcciona al usuario al menú despues de haber guardado su informacion.
Postcondición Se devuelve al menu principal
Excepciones Paso Acción
2 Si se encuentra un perfil creado, el sistema Irá al menú inicial., a continuación este caso de uso queda sin efecto
4 Si el usuario no solicita guardar la informacion ingresada, el sistema arrojará un mensaje diciendo "desea guardar la informaciön"., a continuación este caso de uso continúa
5 Si la información está incompleta, el sistema presenta un mensaje indicando que debe completar la información., a continuación este caso de uso continúa ( retornando al paso 3)
